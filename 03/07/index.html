    Группировка и сортировка по нескольким полям

    Вы уже научились объединять данные в группы и выводить отсортированный результат. SQL позволяет группировать и сортировать данные не только по одному полю, но и по нескольким сразу.

    Группировка по нескольким полям

    В прошлых уроках вы объединяли данные в группы в зависимости от города. Но сформировать группы можно и по двум полям.

    Покажем на примере: нужно сравнить количество заказов по дням, а также учесть, в какой стране был сделан заказ. В этом случае можно сгруппировать данные по двум полям сразу: invoice_date и billing_country.

    Напомним, что поле invoice_date хранит данные типа varchar. Поэтому тип данных лучше сразу преобразовать в тип date. Таблица будет отсортирована по убыванию значений в поле invoice_date.

    SELECT CAST(invoice_date AS date),
       billing_country,
       COUNT(total)
FROM invoice
GROUP BY CAST(invoice_date AS date),
         billing_country 
ORDER BY CAST(invoice_date AS date) DESC
LIMIT 10;
    invoice_date billing_country count
    2013-12-22 India 1
    2013-12-14 Finland 1
    2013-12-09 Portugal 1
    2013-12-06 Canada 1
    2013-12-05 USA 1
    2013-12-04 USA 2
    2013-11-21 USA 1
    2013-11-13 Czech Republic 1
    2013-11-08 Argentina 1
    2013-11-05 Spain 1

    Понять, что с чем группируется, может быть сложно. Сначала сравните прошлую таблицу с результатом запроса с ключевым словом DISTINCT:

    SELECT DISTINCT CAST(invoice_date AS date),
       billing_country
FROM invoice
ORDER BY CAST(invoice_date AS date) DESC
LIMIT 10;
    invoice_date billing_country
    2013-12-22 India
    2013-12-14 Finland
    2013-12-09 Portugal
    2013-12-06 Canada
    2013-12-05 USA
    2013-12-04 USA
    2013-11-21 USA
    2013-11-13 Czech Republic
    2013-11-08 Argentina
    2013-11-05 Spain

    С помощью ключевого слова DISTINCT можно получить похожую выдачу, но без поля с результатом агрегирующей функции COUNT, ведь применить её вместе с DISTINCT не получится. В остальном две таблицы похожи, потому что оператор GROUP BY собрал группы из всех уникальных комбинаций в полях invoice_date и billing_country.

    Сортировка по нескольким полям

    Сортировка по нескольким полям устроена сложнее, чем группировка. При сортировке важен порядок полей, которые указывают после оператора ORDER BY. В случае с группировкой порядок полей влияет только на их расположение в таблице.

    Пока звучит сложно, поэтому приведём пример. Такой запрос выгрузит несколько полей: страну заказа, идентификатор пользователя и сумму заказа. В таблицу попадут заказы только из двух стран: Индии и Португалии.

    SELECT billing_country,
       customer_id,
       total
FROM invoice
WHERE billing_country = 'India' or billing_country = 'Portugal';

    Сортировать можно не только числа, но и строки: страна заказа будет отсортирована в лексикографическом порядке. Идентификатор пользователя будет отсортирован по возрастанию, а сумма заказа — по убыванию.

    SELECT billing_country,
       customer_id,
       total
FROM invoice
WHERE billing_country = 'India' or billing_country = 'Portugal'
ORDER BY billing_country, 
         customer_id,
         total DESC;
    billing_city customer_id total
    India 58 13.86
    India 58 8.91
    India 58 5.94
    India 58 3.96
    India 58 1.99
    India 58 1.98
    India 58 1.98
    India 59 13.86
    India 59 8.91
    India 59 5.94
    India 59 3.96
    India 59 1.99
    India 59 1.98
    Portugal 34 13.86
    Portugal 34 10.91

    Взгляните внимательно на выдачу: сортировка разных полей не конфликтует между собой. Записи первого поля упорядочены в лексикографическом порядке, записи второго — по возрастанию, а записи третьего — по убыванию. Ни одно правило сортировки не нарушает предыдущего.

    Порядок полей действительно важен, ведь первым отсортировано поле со страной заказа. Посмотрите, что получится, если немного изменить запрос.

    SELECT total,
       billing_country,
       customer_id
FROM invoice
WHERE billing_country = 'India' or billing_country = 'Portugal'
ORDER BY total DESC, 
         billing_country,
         customer_id;
    total billing_city customer_id
    13.86 India 58
    13.86 India 59
    13.86 Portugal 34
    13.86 Portugal 35
    10.91 Portugal 34
    8.91 India 58
    8.91 India 59
    8.91 Portugal 35
    5.94 India 58
    5.94 India 59
    5.94 Portugal 34
    5.94 Portugal 35
    3.96 India 58
    3.96 India 59
    3.96 Portugal 34

    Порядок, в котором указывают поля после ORDER BY, влияет на выдачу. Сначала отсортирована сумма заказа по убыванию. Затем для каждого значения суммы отсортированы страны заказа в лексикографическом порядке. Для каждой страны отсортированы значения идентификаторов покупателей от меньшего к большему.

    Обратите внимание, что порядок полей после оператора SELECT влияет только на расположение полей в таблице. Если после SELECT указать поля в другой последовательности, а порядок сортировки оставить прежним, то поля поменяются местами в таблице, но иерархия сортировки не изменится.
