    Группировка данных

    В прошлом уроке вы применяли агрегирующие функции к срезам данных. Например, такой запрос выведет среднюю сумму заказов в Осло:

    SELECT ROUND(AVG(total))
FROM invoice
WHERE billing_city = 'Oslo';

    А если нужно посчитать среднюю сумму заказов для каждого города? Использовать срезы неудобно: придётся писать много запросов. А перед этим нужно получить список уникальных городов, например, с помощью DISTINCT.

    Для таких задач подойдёт группировка. Чтобы разделить данные на группы по значениям полей, в SQL используют оператор GROUP BY. В запросе GROUP BY всегда располагается после условного оператора WHERE. Если WHERE в запросе нет, оператор GROUP BY нужно указать после FROM.

    Теперь можно выгрузить среднюю сумму заказов для каждого города с помощью группировки.

    -- нужно указать поля и необходимые вычисления
SELECT billing_city,
       ROUND(AVG(total))
FROM invoice -- не забыть таблицу
GROUP BY billing_city -- здесь указывают поле, по которому группируют данные
LIMIT 5;
    billing_city round
    Porto 5
    Budapest 7
    Reno 5
    London 5
    Paris 6

    Быстро и красиво. Группировка оператором GROUP BY устроена так: сначала группируются данные по одному городу, для них вычисляется среднее и результат помещается в таблицу. И так для каждого города. Похоже на срезы, только запросов меньше.

    Как вы думаете, что выгрузит такой запрос?

    SELECT billing_city
FROM invoice
GROUP BY billing_city;

    • Ничего, появится сообщение об ошибке.

    • Запрос выгрузит все уникальные значения в поле billing_city.

    • Появится пустая таблица.

    • Появится таблица с количеством значений в каждой группе.

    Что выведет такой запрос?

    SELECT SUM(total)
FROM invoice
GROUP BY billing_city;

    • Вот сейчас точно появится сообщение об ошибке.

    • Категорий в выгрузке нет, поэтому появится пустая таблица.

    • Появится таблица с количеством значений в каждой группе.

    • В таблице будет отображена общая сумма заказов для каждой категории, но категорий не будет.

    Ответы на задания

    Как вы думаете, что выгрузит такой запрос?

    SELECT billing_city
FROM invoice
GROUP BY billing_city;

    • (-) Ничего, появится сообщение об ошибке.

      • Такой запрос сработает.

      • (+) Запрос выгрузит все уникальные значения в поле billing_city.

      • Такая группировка действительно сработает. Запрос выгрузит все уникальные значения в поле. Похоже на работу ключевого слова DISTINCT.

      • (-) Появится пустая таблица.

      • Такая группировка сработает, и таблица не будет пустой.

      • (-) Появится таблица с количеством значений в каждой группе.

      • В запросе нет агрегирующей функции, поэтому подсчёта значений не будет. Такой запрос выгрузит все уникальные значения в поле.

    Что выведет такой запрос?

    SELECT SUM(total)
FROM invoice
GROUP BY billing_city;

    • (-) Вот сейчас точно появится сообщение об ошибке.

      • Запрос выгрузит таблицу, но она будет не очень информативной.

      • (-) Категорий в выгрузке нет, поэтому появится пустая таблица.

      • Таблица не будет пустой: запрос выгрузит сгруппированные данные без категорий.

      • (-) Появится таблица с количеством значений в каждой группе.

      • SUM находит сумму значений, а не их количество. Такой запрос выгрузит таблицу с общей суммой заказов для каждой категории, но самих категорий не будет.

      • (+) В таблице будет отображена общая сумма заказов для каждой категории, но категорий не будет.

      • В таблице действительно не будет категорий. Такая таблица не очень информативна: непонятно, какой категории соответствует результат.
